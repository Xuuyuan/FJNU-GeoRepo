## 1. 虚拟现实（VR）

### 1.1 概念

虚拟现实（Virtual Reality, VR）是以计算机为基础，综合三维图形、多媒体、传感器、跟踪定位和人机交互等技术，创建集视觉、听觉、触觉等多感官体验于一体的虚拟环境，使用户能够借助输入输出设备与虚拟对象进行实时交互，并产生身临其境感觉的综合技术。

### 1.2 本质

虚拟现实本质上是一种先进的人机接口技术。它把人和计算机之间的交流从键盘、鼠标等传统方式，拓展为语音、动作、眼神、姿态、触觉等更自然的感知交互方式。

### 1.3 三大特征（3I）

| 特征 | 英文 | 含义 | 答题关键词 |
| --- | --- | --- | --- |
| 沉浸感 | Immersion | 用户感觉自己进入了虚拟环境 | 三维视觉、空间声音、触觉反馈、身临其境 |
| 交互性 | Interaction | 用户能实时操作虚拟对象并获得反馈 | 实时响应、自然交互、操作对象 |
| 构想性/想象性 | Imagination | 用户在虚拟环境中观察、操作、推演并形成新认知 | 认知、构思、设计、模拟实验 |

### 1.4 VR 系统组成

1. 显示子系统：向用户输出视觉、听觉、触觉等信息，如头盔显示器、立体屏幕、音响、力反馈设备。
2. 检测/监测子系统：接收用户输入或跟踪用户状态，如键盘、鼠标、手柄、数据手套、位置跟踪器、眼动仪。
3. 模拟子系统：负责虚拟环境的建模、渲染、物理模拟、行为模拟和交互逻辑。

### 1.5 VR 模拟对象的三种类型

1. 真实世界的模拟：校园、城市、建筑、小区、旅游景区。
2. 主观虚构环境的模拟：《桃花源记》、大观园、神话故事、科幻场景。
3. 真实存在但肉眼不可见环境的模拟：病毒感染过程、黑洞、引力波、微观结构、天体运行。

### 1.6 VR 与相近概念区别

| 概念 | 重点 | 与 VR 的区别 |
| --- | --- | --- |
| 仿真 | 参数计算、过程推演 | 仿真偏计算表达，VR 偏感知体验和交互呈现 |
| 动画 | 固定路径、固定情节 | 动画是线性播放，VR 支持多路径、多结果、实时交互 |
| 可视化 | 揭示数据规律和本质 | 可视化重表达与分析，VR 更强调沉浸和逼真场景 |
| 虚幻 | 脑中想象，难以直接感知 | 虚拟是计算机可表达、可观看、可交互的对象 |

### 1.7 VR 发展要点

VR 又称虚拟实境、灵境。了解即可：VR 起源与计算机图形学、三维跟踪、仿真、多媒体等技术相关，2016 年常被称为“虚拟现实元年”。

## 2. 增强现实（AR）

### 2.1 概念

增强现实（Augmented Reality, AR）是以真实世界为基础，将计算机生成的文字、图像、三维模型、声音等虚拟信息实时叠加到真实场景中，从而增强用户对现实环境感知和理解的技术。

### 2.2 特征

1. 真实世界为主，虚拟信息为辅。
2. 虚拟对象与现实场景实时融合。
3. 支持用户与真实对象、虚拟对象之间的交互。
4. 强调虚实空间位置的一致性。
5. 依赖多媒体、三维建模、实时视频显示与控制、多传感器融合、实时跟踪注册和场景融合等技术。

### 2.3 AR 关键技术

| 技术 | 含义 | 重点 |
| --- | --- | --- |
| 三维注册/跟踪技术 | 确定观察者、真实对象和虚拟对象的位置与姿态 | 虚实对齐，是 AR 的核心 |
| 标定技术 | 建立真实对象与虚拟信息之间的对应关系 | 特征点、边缘、目标识别 |
| 交互技术 | 用户对虚实对象进行操作 | 手势、触控、语音、体感 |
| 显示技术 | 把虚拟信息叠加到现实视野 | 手机、眼镜、头盔、投影 |

AR 显示设备可分为头盔显示器和非头盔显示设备。透视式头盔显示器又可分为视频透视式和光学透视式，了解即可。

### 2.4 AR 应用

- 教育：实验教学、技能训练、特殊教育。
- 医疗：虚拟手术、解剖显示、术前规划。
- 出行：实景导航、车载 AR、无人驾驶辅助。
- 购物：虚拟试衣、家具摆放、商品展示。
- 工业：装配指导、维修提示。
- 军事：训练、态势感知、目标识别。

## 3. 混合现实（MR）

### 3.1 概念

混合现实（Mixed Reality, MR）是在真实世界与虚拟世界之间建立更深层次融合的技术。它不仅叠加虚拟对象，还要求虚拟对象能够与真实环境发生空间、光照、遮挡和交互关系，形成虚实边界模糊的沉浸体验。

### 3.2 MR 三大技术要求

1. 实时交互：用户操作后系统立即反馈。
2. 自然交互：通过语音、手势、触觉等符合人类习惯的方式操作。
3. 无缝融合：虚拟对象与真实场景在位置、光照、遮挡、比例上保持协调。

### 3.3 VR、AR、MR 对比

| 项目 | VR | AR | MR |
| --- | --- | --- | --- |
| 环境基础 | 虚拟环境 | 真实环境 | 真实与虚拟融合 |
| 虚拟比例 | 高 | 低到中 | 中等且深度融合 |
| 用户感受 | 进入虚拟世界 | 现实世界被增强 | 虚实对象共存并互动 |
| 关键问题 | 沉浸与交互 | 三维注册与标定 | 虚实融合、遮挡、实时交互 |
| 例子 | VR 游戏、虚拟校园 | 手机 AR 导航 | HoloLens 工业维修 |

### 3.4 MR 交互界面类型

- TUI：可触摸用户界面，借助物理控件操作虚拟场景。
- MUI：触摸式/触控用户界面，如手机、平板触控。
- 3DUI：三维用户界面，在三维空间中用手势、语音操作。
- 多通道用户界面：综合视觉、听觉、触觉等通道。
- 混合用户界面：融合多种交互形式。

## 4. 地学可视化与三维 GIS

### 4.1 地学可视化

地学可视化是利用图形、图像、动画、三维模型和交互技术，把地理现象、地学数据和地理知识以直观形式表达出来，以支持地学认知、分析、预测和决策。

地学可视化可分为科学计算可视化和知识可视化。科学计算可视化侧重把科学数据转化为图形图像，用于科研分析和过程模拟；知识可视化侧重把知识结构以图形图表呈现，用于教学、记忆和知识管理。

地理信息可视化的表现形式包括地图、多媒体地学信息、三维仿真地图、虚拟现实等；从维度上可分为二维可视化、三维可视化和多维动态可视化。

### 4.2 地理知识图

地理知识图是把地理概念、地理要素、空间关系、过程关系和属性关系用图形化、网络化方式组织起来的知识表达形式。它属于知识可视化的一种，作用是帮助理解复杂地理知识结构，常用于教学、知识组织和地理认知分析。

### 4.3 地学认知对象

1. 地下：地下管线、地铁、地下停车场、水资源、地质结构。
2. 地表：道路、河流、植被、建筑、地貌。
3. 地上：天气、大气流动、空气质量、云层变化。

### 4.4 地理信息可视化三大模型

| 模型 | 含义 | 例子 |
| --- | --- | --- |
| 对象模型 | 把地理实体看作边界明确的对象 | 建筑物、道路、POI |
| 网络模型 | 强调节点与连线、路径和连通关系 | 交通网络、河网、管线 |
| 场模型/域模型 | 表达连续分布的空间现象 | 温度、降雨、坡度、污染浓度 |

对象模型、网络模型和场模型是复习重点；更细的图斑、等值线、采样等表达方式了解即可。

### 4.5 三维地理信息系统（3D GIS）

三维 GIS 是在传统 GIS 基础上，引入三维空间数据、三维建模和三维可视化技术，对地理空间对象进行采集、管理、分析、表达和交互的一体化系统。

### 4.6 3D GIS 关键技术

- 三维数据获取：遥感、GPS、全站仪、无人机、摄影测量、激光雷达、地图数字化。
- 三维建模：地形建模、建筑建模、道路建模、场景建模。
- 三维分析：距离、面积、体积、坡度坡向、剖面、可视域等。
- 三维可视化：渲染、立体显示、动态表达、交互浏览。
- 多源数据融合：遥感影像、DEM、矢量数据、纹理、属性数据。

### 4.7 3D GIS 与 VR GIS

| 项目 | 3D GIS | VR GIS |
| --- | --- | --- |
| 重点 | 空间数据管理、分析、表达 | 沉浸体验、实时交互、虚拟环境模拟 |
| 技术基础 | GIS、数据库、空间分析 | VR、三维图形、人机交互 |
| 用户体验 | 三维地图浏览和分析 | 进入场景，像在真实环境中行动 |
| 应用 | 城市管理、资源环境、规划 | 虚拟城市、虚拟旅游、灾害演练 |

## 5. 虚拟地理环境（VGE）

### 5.1 概念

虚拟地理环境（Virtual Geographic Environment, VGE）是基于现实地理环境或地理想象环境，在计算机中对地理要素、地理过程、人类活动和社会文化现象进行三维化、动态化、交互化模拟的虚拟环境。它是虚拟现实技术在地理科学中的应用，也被看作继地图之后新的地理学表达语言。

### 5.2 主要特征

1. 空间数据的真实表现：真实再现地形、环境等空间数据，也可模拟过去、未来或不存在的事物。
2. 任意角度观察与实时交互：用户可观察、浸入、移动，并与虚拟环境互动。
3. 具备基于三维空间数据库的 GIS 功能：如查询、空间分析等。
4. 可视化是自然完整的用户接口：三维可视化不仅是展示结果，也是用户进入和操作系统的界面。

### 5.3 层面结构

虚拟地理环境由实景面和虚景面构成。

| 层面 | 所属 | 含义 |
| --- | --- | --- |
| 地理位置层面 | 实景面 | 确定对象在地理空间中的位置 |
| 内表达数据层面 | 实景面 | 地理数据、属性数据、模型数据 |
| 外表达境界/外表达体线层面 | 实景面 | 图形图像、三维场景、可视化表达 |
| 单主体感知认知层面 | 虚景面 | 单个用户对虚拟环境的感知和理解 |
| 互主体社会层面 | 虚景面 | 多用户之间的交流、协同和社会行为 |

### 5.4 发展阶段

1. 虚拟群体社会：早期网络群体交流与虚拟社区。
2. 虚拟村落社会：规模扩大，具有较稳定的空间组织。
3. 虚拟城市社会：功能复杂，具备社会、经济、文化、管理等综合属性。

### 5.5 虚拟地理环境系统概念

虚拟地理环境系统是基于网络或计算机平台的多用户虚拟三维环境，可用于发布地学多维数据，模拟和分析复杂地学现象过程，支持地学数据解释、未来场景预见、设计规划和决策等，也可用于教育、旅游和娱乐。其三维空间既可以对应现实地理空间，也可以是协同工作和群体决策的虚拟地理试验空间。

### 5.6 VGE 关键技术

| 技术 | 作用 |
| --- | --- |
| 三维数据获取 | 获取地形、建筑、高度、纹理、属性等数据 |
| 三维建模 | 构建地形、建筑、道路、水体、植被、人物等模型 |
| 人机交互 | 支持用户漫游、选择、控制、查询和反馈 |
| 三维可视化 | 以图形、动画、立体显示呈现地理信息 |
| 虚拟现实引擎 | 支撑渲染、物理、动画、脚本、交互 |
| 三维全景 | 通过多角度拍摄与拼接展示真实场景 |
| 硬件设备 | 显示、跟踪、输入、输出、力反馈等 |

### 5.7 三维建模类型

1. 几何建模：描述对象形状结构，常见有线框模型、表面模型、实体模型。
2. 物理建模：模拟重力、碰撞、摩擦、流体等物理规律。
3. 行为建模：描述对象动作、状态变化和智能行为。
4. 粒子建模：模拟火焰、烟雾、雨雪、瀑布、落叶等大量离散粒子效果。

### 5.8 人机交互发展阶段

1. 字符显示器交互阶段：命令行输入。
2. 图形显示器交互阶段：窗口、鼠标、图形界面。
3. 多媒体技术交互阶段：图像、声音、视频等多媒体。
4. 多模态自然交互阶段：语音、手势、体感、眼动、脑机接口等。

### 5.9 主动输入与被动输入

| 类型 | 含义 | 例子 |
| --- | --- | --- |
| 主动输入 | 用户主动发出操作命令 | 键盘、鼠标、按钮、手柄点击 |
| 被动输入 | 系统主动监测用户状态并触发反馈 | 头部跟踪、手部位置跟踪、眼动跟踪 |

### 5.10 跟踪定位与常用输入设备

跟踪定位技术用于及时准确地获取人的动态位置和方向信息，并将该位置和方向信息发送到虚拟现实计算机控制系统中。课堂笔记强调两种方式：一是跟踪人的头部位置与方向，用来确定用户视点与视线方向；二是跟踪用户手的位置与方向，用来确定手部操作和交互动作。六自由度包括三个平移自由度和三个旋转自由度。

基于物理设备跟踪的输入设备主要包括：

1. 道具：方向盘、手术器械模型、枪械模型等。
2. 物理控制器：手柄、数据手套、空间控制器。
3. 平台：运动平台、驾驶舱、跑步平台。

### 5.11 VGE 系统类型与常用平台

虚拟地理环境系统常见类型包括沉浸式、桌面式、分布式和移动式。沉浸式系统强调头盔、洞穴式显示等带来的沉浸体验；桌面式系统依托普通计算机进行三维浏览和交互；分布式系统支持多用户、网络化协同；移动式系统面向手机、平板、一体机等移动终端。常用平台和工具包括三维可视化系统、BIM/3ds Max 等建模工具、Unity/Unreal Engine 等游戏引擎，以及 VRML 等底层造型语言。

## 6. 三维可视化、立体显示与全景技术

### 6.1 视差

视差是由于观察位置不同而导致同一物体在视网膜或图像中的位置差异。人能够产生立体视觉的重要原因是双眼之间存在生理视差。

### 6.2 立体显示方式

| 方式 | 原理 |
| --- | --- |
| 场分割显示 | 左右眼图像按时间或帧交替显示 |
| 分色显示 | 用不同颜色通道分离左右眼图像 |
| 分光立体显示 | 通过偏振、光谱等方式分离左右眼图像 |

### 6.3 全息技术

全息技术是记录和再现物体光波全部信息的显示技术，能够表现物体的三维空间感。复习时掌握“记录光波信息、再现三维影像”即可。

### 6.4 虚拟现实引擎技术

虚拟现实引擎是支撑虚拟场景构建、渲染、动画、物理模拟、脚本控制、交互和发布的平台。常见引擎包括 Unity、Unreal Engine 等。

### 6.5 三维全景技术

三维全景技术通过对现实场景进行多角度拍摄，再进行图像拼接和交互浏览，使用户能够在固定或半固定位置向各个方向观察真实场景。特点是制作相对方便、真实感强、适合虚拟旅游、房产展示、校园展示等。

## 7. 虚拟地理环境应用

### 7.1 主要应用领域

## 9. 数字地球、数字城市、智慧城市

### 9.1 数字地球

数字地球是把地球及其相关自然、人文、经济、社会信息，以多分辨率、多尺度、多时空和三维方式进行数字化表达和管理的综合信息系统。它强调全球尺度、多源数据集成和地球系统表达。

### 9.2 数字城市

数字城市是以城市空间信息为核心，利用计算机、多媒体、大规模存储、宽带网络、3S 技术和虚拟现实技术，对城市进行多分辨率、多尺度、多时空、多类型的数字化构建与表达。它是真实城市的数字化映射。

城市建模对象包括地形地貌、建筑物、道路、桥梁、树木、汽车、人物等。城市建模一般包括明确建模目的和任务、准备并处理数据、进行模型分解与组织、完成建模与组合等步骤。常用数据包括地形数据、数字化地图数据、建筑物高度数据、航拍影像、地面近景照片、纹理图片、属性数据和三维建筑几何数据。

### 9.3 智慧城市

智慧城市是在数字城市基础上，进一步利用物联网、传感器、大数据、人工智能、云计算、GIS 和网络通信技术，对城市运行进行感知、传输、处理、分析和智能服务，实现城市管理的信息化、智能化、精细化和动态化。

### 9.4 三者关系

| 概念 | 尺度 | 重点 | 关系 |
| --- | --- | --- | --- |
| 数字地球 | 全球 | 地球系统数字化 | 宏观基础框架 |
| 数字城市 | 城市 | 城市空间信息数字化 | 数字地球在城市尺度的应用 |
| 智慧城市 | 城市 | 感知、分析、智能服务 | 数字城市的进一步发展 |

### 9.5 智慧城市应用

- 智慧交通：拥堵监测、路线诱导、交通控制、出行服务。
- 智慧医疗：医院、社区、药店、血站互联互通。
- 智慧家居：安防、门禁、视频对讲、远程家电控制。
- 智慧建筑：利用感知、控制和管理系统提高建筑运行效率与安全性。
- 电子政务：在线办事、便民服务、公共信息平台。
- 数字旅游：导航、景点介绍、智能酒店、电子菜单。
- 公共安全：平安城市、灾害预警、火灾监测、应急管理。
- 环境生态监测：温度、湿度、光照、水质、污染、辐射。
- 食品安全溯源：生产、运输、销售全过程追踪。
- 智慧物流：仓储、配送、路径优化、状态跟踪。

### 9.6 智慧城市支撑技术与体系结构

支撑技术：

1. 新型感知技术。
2. 网络通信技术。
3. 数据运算与智能处理技术。
4. 城市空间信息技术。

体系结构：

4. 应用层：智慧交通、公共安全、智慧市政、环境监测等。

## 10. VRML 基础

### 10.1 VRML 概念

VRML（Virtual Reality Modeling Language，虚拟现实造型语言）是一种用于在网络环境中描述三维虚拟世界的文本型语言。它通过节点和字段描述几何对象、外观、材质、空间位置、光照、声音、动画和交互。

### 10.2 文件扩展名与文件头

- 文件扩展名：课件标准写法为 `.wrl`。
- 常见文件头：`#VRML V2.0 utf8`
- 含义：说明该文件是 VRML 文件，采用 VRML 2.0 规范，字符集为 UTF-8。
- 注释：以 `#` 开头的内容为注释。

### 10.3 节点与字段

节点是 VRML 中最基本的组成单元。每个节点由节点名称和一对大括号组成，大括号内部是字段和字段值。

```vrml
Shape {
  appearance Appearance {
    material Material {
      diffuseColor 1 0 0
    }
  }
  geometry Box {
    size 2 2 2
  }
}
```

### 10.4 字段值类型

- 单值：一个数值、布尔值、字符串、颜色、坐标等。
- 多值：一组数值、坐标点、颜色、索引等。

### 10.5 DEF 与 USE

- `DEF`：给节点定义名称。
- `USE`：复用已定义的节点。
- 作用：减少重复代码，便于管理同类对象。

```vrml
DEF Tree Shape { ... }
Transform {
  translation 5 0 0
  children [ USE Tree ]
}
```

### 10.6 事件与 ROUTE

- 事件：节点之间传递的信息，可分为入事件和出事件。
- ROUTE：事件传递的通道，把一个节点的输出事件连接到另一个节点的输入事件。
- 常用于动画和交互。

```vrml
ROUTE Touch.touchTime TO Timer.startTime
ROUTE Timer.fraction_changed TO Move.set_fraction
ROUTE Move.value_changed TO Obj.set_translation
```

## 11. VRML 造型与几何节点

### 11.1 Shape 节点

`Shape` 是造型节点，用于创建虚拟对象。它通常包括两个字段：

1. `appearance`：外观特性，包括材质、颜色、贴图、透明度。
2. `geometry`：几何特性，包括对象形状和空间结构。

### 11.2 标准几何体

| 节点 | 作用 | 主要字段 |
| --- | --- | --- |
| Box | 长方体 | `size` 长宽高 |
| Cone | 圆锥 | `bottomRadius`、`height`、`side`、`bottom` |
| Cylinder | 圆柱体 | `radius`、`height`、`side`、`top`、`bottom` |
| Sphere | 球体 | `radius` |

### 11.3 地表网格节点 ElevationGrid

用途：构建地形、地貌、起伏表面。  
原理：把地表划分成规则格网，每个格点赋予高度值，再形成连续地形表面。

常见字段：

- `xDimension`、`zDimension`：X/Z 方向网格点数量。
- `xSpacing`、`zSpacing`：网格间距。
- `height`：每个格点的高度。
- `colorPerVertex`：颜色按顶点还是按面设置。
- `creaseAngle`：控制地形表面平滑程度。
- `solid`、`ccw`：控制渲染方向和面显示。

### 11.4 成型节点 Extrusion

用途：创建复杂拉伸形体。  
原理：把一个二维截面沿某条三维路径拉伸，形成特殊三维模型。

关键字段：

- `crossSection`：截面形状。
- `spine`：拉伸路径。
- `scale`：沿路径不同位置的缩放。
- `orientation`：沿路径的旋转。
- `beginCap`、`endCap`：是否封闭顶盖和底盖。

### 11.5 点线面节点

| 节点 | 作用 | 关键点 |
| --- | --- | --- |
| PointSet | 点集合 | 坐标 + 颜色，可模拟星点、指示灯、无人机阵列 |
| IndexedLineSet | 指针线集合 | 用点坐标和索引连接线段，`-1` 表示一段线结束 |
| IndexedFaceSet | 指针面集合 | 用点坐标和索引构成面，可表达复杂多边形模型 |
| Text | 文本 | 在虚拟世界中显示文字，涉及文字内容、字体、大小、对齐 |

## 12. VRML 外观节点

### 12.1 Appearance 节点

`Appearance` 表示对象外观，常包含三个子节点：

1. `Material`：材质参数。
2. `ImageTexture` / `PixelTexture` / `MovieTexture`：纹理贴图。
3. `TextureTransform`：贴图变换。

### 12.2 Material 材质节点

常见字段：

- `diffuseColor`：漫反射颜色，即物体主要颜色。
- `ambientIntensity`：环境光影响强度。
- `specularColor`：高光颜色。
- `emissiveColor`：自发光颜色。
- `shininess`：光泽度。
- `transparency`：透明度。

### 12.3 贴图节点

| 节点 | 作用 |
| --- | --- |
| ImageTexture | 使用图片作为纹理 |
| PixelTexture | 使用像素矩阵作为纹理 |
| MovieTexture | 使用视频作为纹理 |

`ImageTexture` 常见字段：

- `url`：图片路径。
- `repeatS`：水平方向是否重复。
- `repeatT`：垂直方向是否重复。

### 12.4 TextureTransform 贴图变换

用于控制贴图在几何表面上的位置、旋转、缩放和平移。

字段：

- `center`：旋转或缩放中心。
- `rotation`：旋转角度。
- `scale`：贴图缩放或重复倍数。
- `translation`：贴图平移。

## 13. VRML 群组节点

### 13.1 群组节点作用

群组节点用于组织多个子节点，便于统一管理、移动、旋转、缩放、复用和优化复杂场景。

### 13.2 常见群组节点

| 节点 | 作用 | 可能考法 |
| --- | --- | --- |
| Group | 普通群组，把多个节点组合在一起 | 说明群组作用 |
| Transform | 转换型群组，对子节点进行平移、旋转、缩放 | 解释空间位置控制 |
| Inline | 引入型节点，把外部 `.wrl` 文件引入当前场景 | 第 12 周强调，可能简答 |
| Switch | 选择型节点，在多个子节点中选择一个显示 | 可用于状态切换 |
| Billboard | 广告牌节点，使对象始终朝向观察者 | 标识牌、树木贴片 |
| Anchor | 超链接节点，点击对象跳转到其他文件或网页 | 场景链接 |

### 13.3 Inline 引入型节点答题模板

Inline 节点用于把外部 VRML 文件引入当前场景。它可以把复杂场景拆分成多个小文件，例如地形、建筑、树木、人物分别建模，再通过 Inline 统一调用。这样有利于减少单个文件复杂度，提高代码复用性，便于分工开发、管理维护和系统扩展。

## 14. VRML 环境节点

### 14.1 Background 背景节点

作用：设置虚拟世界背景，包括天空、大地颜色或六面贴图背景。

常见字段：

- `skyColor`、`skyAngle`：天空颜色和角度。
- `groundColor`、`groundAngle`：地面颜色和角度。
- `frontUrl`、`backUrl`、`leftUrl`、`rightUrl`、`topUrl`、`bottomUrl`：六面背景贴图。

### 14.2 Fog 雾化节点

作用：模拟雾、烟、远处模糊等环境效果，增强空间深度。

字段：

- `color`：雾颜色。
- `visibilityRange`：可见距离。
- `fogType`：雾化类型，如线性或指数。

### 14.3 光照节点

| 节点 | 含义 | 用途 |
| --- | --- | --- |
| PointLight | 点光源，向周围发光 | 灯泡、路灯 |
| DirectionalLight | 平行光源，来自无限远方向 | 太阳光 |
| SpotLight | 锥形光源 | 手电筒、舞台灯、车灯 |

常见字段：

- `on`：开关。
- `color`：光颜色。
- `intensity`：光照强度。
- `ambientIntensity`：环境光强度。
- `location`：位置。
- `direction`：方向。
- `radius`：影响范围。
- `attenuation`：衰减方式。
- `beamWidth`、`cutOffAngle`：锥形光范围。

### 14.4 Sound 声音节点

作用：在虚拟环境中加入空间声音。  
声音文件常见格式：`.wav`、`.mid` 等。

相关节点和字段：

- `Sound`：控制声音的位置、方向、前后最大最小传播距离。
- `AudioClip`：指定声音文件、循环、播放速度、开始时间等。
- 距离越远，声音可逐渐衰减，增强空间真实感。

## 15. VRML 观看点、动画与交互

### 15.1 观看点设置

| 节点 | 作用 |
| --- | --- |
| Viewpoint | 设置观察位置、观察方向、视野角度，可切换视点 |
| NavigationInfo | 设置用户化身、头灯、移动速度、浏览方式、可见距离 |

`NavigationInfo` 常见内容：

- 化身尺寸。
- 头灯是否开启。
- 移动速度。
- 浏览模式。
- 最远观看距离。

### 15.2 动态虚拟现实

动态虚拟现实强调对象随时间变化。核心通常包括：

1. 时间传感器：产生时间进度。
2. 插补器：根据关键帧生成中间状态。
3. ROUTE：把时间事件传给插补器，再传给对象字段。

### 15.3 时间传感器 TimeSensor

常见字段：

- `cycleInterval`：一个周期持续时间。
- `loop`：是否循环。
- `startTime`：开始时间。
- `stopTime`：停止时间。

### 15.4 插补器节点

| 插补器 | 作用 |
| --- | --- |
| PositionInterpolator | 位置变化 |
| OrientationInterpolator | 方向/旋转变化 |
| ScalarInterpolator | 标量变化，如强度 |
| ColorInterpolator | 颜色变化 |
| CoordinateInterpolator | 坐标变化 |
| NormalInterpolator | 法线变化 |

关键字段：

- `key`：关键帧时间比例。
- `keyValue`：关键帧对应的值。

### 15.5 交互传感器

| 节点 | 作用 |
| --- | --- |
| TouchSensor | 接触、点击、鼠标移入触发 |
| PlaneSensor | 在平面中拖动对象 |
| CylinderSensor | 绕单轴旋转对象 |
| SphereSensor | 任意方向旋转对象 |
| VisibilitySensor | 判断对象是否进入视野 |
| ProximitySensor | 判断用户是否进入某空间范围 |
| Collision | 碰撞检测 |

## 16. 高频名词解释模板

### 虚拟现实

虚拟现实是以计算机为基础，综合三维图形、多媒体、传感器、跟踪定位和人机交互等技术，创建具有视觉、听觉、触觉等多感官体验的虚拟环境，使用户能够与虚拟对象实时交互并产生身临其境感觉的综合技术。其主要特征包括沉浸感、交互性和构想性。

### 增强现实

增强现实是以真实世界为基础，将计算机生成的虚拟信息实时叠加到真实场景中，从而增强用户对现实环境感知和理解的技术。其关键技术包括三维注册、标定、交互和显示技术。

### 混合现实

混合现实是将真实世界与虚拟对象深度融合，使虚拟对象能够与真实环境共存并发生交互的技术。它强调实时交互、自然交互和无缝融合，处于虚拟现实与增强现实之间。

### 地学可视化

地学可视化是利用图形、图像、动画、三维模型和交互技术，将地学数据、地理现象和地理知识直观表达出来，以支持地学认知、分析、模拟和决策的技术方法。

### 地理知识图

地理知识图是用图形化、网络化方式组织和表达地理概念、地理要素、空间关系、过程关系和属性关系的知识表达形式，属于知识可视化内容，可用于地理教学、知识组织和地理认知分析。

### 三维地理信息系统

三维地理信息系统是在传统 GIS 基础上结合三维空间数据、三维建模、空间分析和三维可视化技术，对地理空间对象进行采集、管理、分析和表达的一体化系统。

### 虚拟地理环境

虚拟地理环境是基于现实或想象地理环境，在计算机中对地理要素、地理过程、人类活动和社会文化现象进行三维化、动态化和交互化模拟的虚拟环境，是虚拟现实技术在地理科学中的重要应用。

### 跟踪定位

跟踪定位是及时准确地获取用户的位置、方向和运动状态，并将其同步到虚拟环境中的技术。它是实现自然交互和沉浸体验的重要基础，课堂重点掌握两种方式：跟踪头部位置与方向、跟踪手部位置与方向。

### 数字地球

数字地球是把地球及其自然、人文、经济和社会信息，以多分辨率、多尺度、多时空和三维方式进行数字化表达和管理的综合信息系统，强调全球尺度、多源数据集成和地球系统表达。

### 数字城市

数字城市是以城市空间信息为核心，利用计算机、网络、3S、三维建模和虚拟现实等技术，对真实城市进行多尺度、多分辨率、多时空的数字化表达和管理的城市信息系统。

### 智慧城市

智慧城市是在数字城市基础上，利用物联网、传感器、大数据、人工智能、GIS 和网络通信技术，对城市运行状态进行感知、传输、处理和智能服务，实现城市管理智能化、精细化和动态化。

### VRML

VRML 是虚拟现实造型语言，是一种用于描述三维虚拟世界的文本型语言。它通过节点和字段表达几何对象、外观、位置、光照、声音、动画和交互，文件扩展名通常为 `.wrl`。

### 节点

节点是 VRML 中最基本的组成单元，每个节点由节点名称和字段组成，用于描述虚拟世界中的对象、外观、位置、声音、动画或交互等内容。

### 路由

路由是 VRML 中事件传递的通道，用于把一个节点产生的输出事件传递给另一个节点的输入事件，从而实现动画、交互和对象状态变化。

## 17. 高频简答题

### 17.1 简述 VR、AR、MR 的区别

VR 以计算机生成的虚拟世界为主体，用户主要沉浸在虚拟环境中；AR 以真实世界为基础，把虚拟信息叠加到真实场景中，用于增强现实感知；MR 则进一步强调真实对象与虚拟对象之间的深度融合和实时交互。三者的核心差异在于真实世界和虚拟世界所占比例不同，以及虚实融合深度不同。

### 17.2 简述虚拟现实的三大特征

虚拟现实具有沉浸感、交互性和构想性。沉浸感指用户通过视觉、听觉、触觉等感官刺激产生身临其境的感觉；交互性指用户能够对虚拟对象进行实时操作并获得反馈；构想性指用户在虚拟环境中观察、操作和推演，从而加深认知并形成新的设计或理解。

### 17.3 简述增强现实的关键技术

增强现实关键技术包括三维注册技术、标定技术、交互技术和显示技术。三维注册用于确定观察者、真实对象和虚拟对象的位置姿态，实现虚实对齐；标定技术用于建立真实目标与虚拟信息之间的对应关系；交互技术支持用户操作虚实对象；显示技术负责把虚拟信息叠加到真实场景中。

### 17.4 简述虚拟地理环境的层面结构

虚拟地理环境包括实景面和虚景面。实景面包括地理位置层面、内表达数据层面和外表达境界层面，主要负责地理对象的位置、数据和可视化表达。虚景面包括单主体感知认知层面和互主体社会层面，强调用户对虚拟环境的感知、认知，以及多用户之间的交流协同和社会行为。

### 17.5 简述虚拟地理环境关键技术

虚拟地理环境关键技术包括三维数据获取、三维建模、人机交互、三维可视化、虚拟现实引擎、三维全景和硬件支撑等。三维数据获取提供地形、建筑、纹理和属性数据；三维建模构建场景对象；交互技术支持用户操作和漫游；三维可视化和引擎技术负责表达、渲染和运行；硬件设备提供沉浸体验和输入输出支持。

### 17.6 简述三维建模中几何建模的三种模型

几何建模常见包括线框模型、表面模型和实体模型。线框模型用点和线表达物体骨架，结构简单但缺少表面信息；表面模型用面片表达物体外表，适合可视化表达；实体模型不仅表达外表，还表达内部体积和实体属性，适合空间分析和物理模拟。

### 17.7 简述主动输入与被动输入的区别

主动输入是用户主动发起的操作，如键盘输入、鼠标点击、手柄按钮等；被动输入是系统通过传感器主动监测用户位置、姿态、眼动或动作变化，并将这些变化作为输入。主动输入强调用户明确命令，被动输入强调系统感知用户状态。

### 17.8 简述 VRML 中 Inline 节点的作用

Inline 节点用于把外部 VRML 文件引入当前虚拟场景。它可以把复杂场景拆分为多个小文件，如地形、建筑、植被、人物分别存储，再在主文件中统一调用。这样便于代码复用、分工开发、文件管理和系统维护，也能降低单个文件的复杂度。

### 17.9 简述 VRML 中 DEF 与 USE 的作用

DEF 用于给某个节点定义名称，USE 用于复用已经定义的节点。通过 DEF/USE，可以避免重复书写相同对象，提高代码复用性和可维护性。例如多个相同的树木、路灯或建筑构件可以只定义一次，再在不同位置复用。

### 17.10 简述智慧城市体系结构

智慧城市体系通常包括感知层、网络层、平台层和应用层。感知层通过传感器、摄像头、GPS 等采集城市运行数据；网络层负责数据传输；平台层进行数据存储、处理、挖掘和智能分析；应用层面向智慧交通、公共安全、环境监测、电子政务等提供具体服务。

## 18. 论述题准备

### 18.1 论述虚拟地理环境在智慧城市中的应用

答题思路：

1. 先定义 VGE 和智慧城市。
2. 说明 VGE 可提供三维空间表达、动态模拟、交互分析和沉浸式决策支持。
3. 从智慧交通、公共安全、环境监测、数字旅游、城市规划等方面举例。
4. 说明关键技术：三维数据获取、GIS、遥感、GPS、三维建模、虚拟现实引擎、大数据、传感器。
5. 总结其价值：提高城市管理效率、增强决策直观性、支持应急演练和公众参与。

### 18.2 论述如何设计一个虚拟地理环境系统

答题思路：

设计虚拟地理环境系统，应围绕具体地理问题，把真实地理数据、三维场景、地理过程模型和交互功能结合起来。以我们小组“基于虚拟地理环境的洪水风险交互模拟游戏”为例，可从以下方面论述：

1. 明确目标和用户。系统不是单纯做三维景观，而是面向防灾科普和课堂教学，让用户扮演“防洪决策者”，在虚拟场景中观察洪水、布设堤坝、比较防洪方案。

2. 建立数据基础。利用 DEM、遥感影像、河流、村庄、农田等数据构建真实地理环境。本组系统计划将 DEM 和遥感影像经过裁剪、投影转换、重采样等处理后，导入 Unity 生成三维地形和地表纹理。

3. 设计系统结构。系统可分为数据层、逻辑层、表现层和交互层：数据层管理高程和影像，逻辑层负责洪水扩散和堤坝阻挡，表现层显示三维地形和动态水面，交互层实现漫游、点击、UI 面板和模拟控制。

4. 构建场景和模型。根据地形起伏建立三维地貌，并加入河流、低洼区、村庄、水体、植被和堤坝等要素，使用户能直观判断洪水可能流向和重点保护对象。

5. 实现动态模拟和交互。本组案例采用基于网格的 BFS 洪水扩散模型，洪水从水源点向满足高程和连通条件的网格扩散。用户放置堤坝后，会改变网格之间的阻挡关系，从而影响洪水蔓延范围，形成“观察、决策、模拟、反馈”的闭环。

6. 进行优化和验证。系统开发完成后，需要进行帧率优化、模型简化、贴图压缩和功能测试；还可将洪水淹没范围与专业软件结果对比，并通过用户反馈评价科普效果。

总之，虚拟地理环境系统的设计重点是“真实数据支撑、三维可视表达、动态过程模拟、用户交互反馈”。结合我们的小组系统，就是用 DEM 和遥感影像搭建洪水场景，用 BFS 模型模拟洪水扩散，用堤坝布设和评分反馈帮助用户理解防洪决策。

### 18.3 论述 VRML 在虚拟现实系统开发中的作用

答题思路：

1. VRML 是文本型虚拟现实造型语言，用于描述三维虚拟世界。
2. 它以节点为基本单元，可表达几何体、外观、材质、贴图、背景、光照、声音等。
3. 通过 Transform、Group、Inline 等节点组织复杂场景。
4. 通过 TimeSensor、Interpolator、ROUTE 实现动画。
5. 通过 TouchSensor、PlaneSensor、ProximitySensor 等实现交互。
6. 学习 VRML 有助于理解虚拟环境底层构建原理，适合完成小型虚拟现实系统。

- VR 三大特征：沉浸感、交互性、构想性。

- AR 关键技术：三维注册、标定、交互、显示。

- 地理信息可视化三大模型：对象模型、网络模型、场模型。

- VGE 五个层面：地理位置层面、内表达数据层面、外表达境界层面、单主体感知认知层面、互主体社会层面。

- VGE 系统类型：沉浸式、桌面式、分布式、移动式。
- VGE 应用：数字地球、数字城市、数字地域、数字流域、虚拟城市规划、虚拟旅游、虚拟地理实验、虚拟教学、商业应用。

- 三维 GIS 分析：空间查询、属性计算、距离/体积测算、坡度坡向、等值线、剖面、可视域。

- 跟踪定位两种方式：跟踪头部位置与方向、跟踪手部位置与方向。
- VGE 系统构成：虚拟世界、硬件接口、软件接口。

- VRML 注释符号：`#`。

- 群组节点：Group、Transform、Inline、Switch、Billboard、Anchor。
